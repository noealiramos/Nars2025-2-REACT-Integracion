# Hero width fix

## Problema detectado

- En `Home`, el bloque hero mostraba un hueco visible a la derecha del texto.
- La causa no era un layout de dos columnas ni un contenedor derecho vacio.
- La causa raiz era una restriccion local en `ecommerce-app-Nars/src/pages/HomePage.css`:

```css
.home-hero__text {
  margin: 0;
  max-width: 46rem;
  color: #cbd5e1;
}
```

- Como `.home-hero` ya ocupa el ancho del contenedor y no tiene segunda columna, ese `max-width` dejaba espacio libre a la derecha en tablet/desktop.

## Causa raiz

- Restriccion de ancho innecesaria en `.home-hero__text`.
- No habia problema adicional de grid multi-columna, `justify-between` o columna vacia.

## Codigo antes/despues

### Antes

```css
.home-hero__text {
  margin: 0;
  max-width: 46rem;
  color: #cbd5e1;
}
```

### Despues

```css
.home-hero__text {
  margin: 0;
  max-width: none;
  color: #cbd5e1;
}
```

## Justificacion tecnica

- Es el cambio minimo y mas seguro.
- No toca estructura JSX.
- No cambia el layout general del hero.
- No afecta otros componentes porque la regla es especifica de `HomePage.css`.
- Mantiene comportamiento responsive: en mobile ya no habia restriccion efectiva y en desktop/tablet el texto ahora puede usar todo el ancho disponible del hero.

- `ecommerce-app-Nars/src/pages/HomePage.css`

## Resultados

- El texto del hero queda habilitado para ocupar todo el ancho disponible del bloque.
- Tecnica y estructuralmente, el hueco derecho deberia desaparecer porque la unica restriccion detectada era `max-width: 46rem`.
- No se detectan impactos colaterales por build.

## Validacion responsive

- `mobile`: sin riesgo aparente; ya existia override movil compatible.
- `tablet`: mejora esperada, porque el texto deja de quedarse corto.
- `desktop`: mejora esperada, porque desaparece la limitacion artificial del ancho.

## Observacion

- No se realizo validacion visual interactiva en navegador desde esta sesion, asi que la confirmacion final visual queda inferida por la estructura real del CSS/JSX y la ausencia de layout doble.
