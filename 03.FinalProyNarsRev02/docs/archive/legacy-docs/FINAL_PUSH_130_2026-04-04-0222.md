  - Mitigación: se mantuvieron los mismos servicios/endpoints y solo se encapsularon mutations + invalidación.

# 5. Implementación realizada

## useMutation real y visible

- `useWishlistActions` usa `useMutation` para:
  - agregar a wishlist
  - quitar de wishlist
  - limpiar wishlist
  - mover a carrito
- `useAdminCategories` usa `useMutation` para:
  - crear categoría
  - editar categoría
  - eliminar categoría

En ambos casos hay invalidación coherente vía React Query.

## Custom hooks reales

Se implementaron 3 hooks reales, no triviales:

- `useWishlistActions`
  - encapsula mutations, invalidación y mensajes UI
- `useAdminCategories`
  - encapsula query, mutations, refetch e invalidación
- `useCartStockValidation`
  - encapsula lógica de stock válida para add/update y mensajes de error asociados

## Validación estricta de stock en carrito

- `CartContext` ahora valida cantidades usando `useCartStockValidation`.
- Se mapea `stock` desde backend al estado del carrito.
- Se impide:
  - cantidad inválida
  - superar stock disponible
  - agregar productos agotados
- `CartItem` muestra `Stock: X` y bloquea el botón `+` cuando ya se alcanzó el máximo.

## Estados de carga/error/empty más ricos

Se mejoraron al menos 3 vistas:

- `WishlistPage`
  - loading, error y empty state más defendibles
- `AdminCategoriesPage`
  - loading, error con retry y estado vacío más claro
- `HomePage`
  - loading, error con retry y empty state para búsquedas sin resultados

## Responsive mínimo

- Se reforzó wishlist y admin categories para que no colapsen en móvil:
  - acciones flexibles
  - layout de tarjetas más robusto en ancho reducido

## Causa raíz y corrección

- Falla de tests:
  - `CartContext.integration.test.jsx` montaba `CartProvider` sin `UIProvider`.
  - Se corrigió envolviendo el wrapper con `UIProvider`.
  - Además el mock del producto no incluía `stock`, por lo que la nueva validación bloqueaba el update; se corrigió agregando `stock` al mock.

- Falla de Cypress:
  - `cart.cy.js` asumía cambios exactos de cantidad con un estado demasiado rígido respecto al stock persistido real.
  - Se corrigió el spec para usar una sanity más robusta: interacción real con cantidad y luego vaciado del carrito sin depender de un número intermedio frágil.
