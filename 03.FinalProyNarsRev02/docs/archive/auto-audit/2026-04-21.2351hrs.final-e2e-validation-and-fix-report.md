# Informe Final — Validación E2E y Corrección de Bug de Carrito

## 1. Resumen ejecutivo

Se identificó y corrigió un bug crítico en el flujo de carrito relacionado con la validación de stock.

El problema se originaba por una desalineación entre frontend y backend, donde el backend no enviaba el campo `stock` en los productos del carrito y el frontend interpretaba su ausencia como `0`, generando falsos mensajes de agotado.

Se implementó un fix mínimo y controlado que alinea el contrato de datos y endurece la validación del frontend.

Posteriormente, se validó el sistema completo mediante pruebas unitarias, de integración y E2E.

---

## 2. Problema identificado

- Mensaje falso: "Este producto ya no tiene stock disponible"
- Bloqueo de incremento y decremento en carrito
- Stock real disponible en base de datos

---

## 3. Causa raíz

Desalineación FE/BE:

- Backend no incluía `stock` en:
- Frontend convertía `undefined` → `0`
- Validación interpretaba `0` como agotado

---

## 4. Solución implementada

### Backend
- Se agregó `stock` al populate del carrito

### Frontend
- Se evitó convertir ausencia de stock a `0`
- Se implementó validación diferenciada:
- dato faltante → no bloquear
- stock = 0 → bloquear

## Comparativa antes vs después

Antes del fix:
- El carrito interpretaba stock faltante como 0
- Se mostraba mensaje falso de "sin stock"
- No permitía incrementar ni decrementar

Después del fix:
- El backend envía stock correctamente
- El frontend distingue entre dato faltante y stock real
- Se permite modificar cantidades correctamente
- El mensaje de "sin stock" solo aparece cuando corresponde

## Impacto del cambio

El fix impacta únicamente el flujo de carrito y su validación de stock.

No se afectaron:
- Checkout
- Órdenes
- Autenticación
- Renderizado de productos

Esto fue validado mediante pruebas E2E completas.
