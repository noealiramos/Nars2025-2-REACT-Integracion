## 3. UX implementada

La UX observada y validada cumple con lo pedido:

- si existen direcciones guardadas, se muestran opciones reutilizables y una opcion `Usar una dirección nueva`
- si existen metodos de pago guardados, se muestran opciones reutilizables y una opcion `Usar un método nuevo`
- si el usuario selecciona una opcion existente, los campos relacionados quedan bloqueados y se usa directamente ese recurso
- si el usuario selecciona `nuevo`, reaparece la captura normal
- si no existen datos previos, el flujo opera con el formulario normal
- no se mezclan datos seleccionados con datos del formulario: la logica se decide con `shippingMode`, `paymentMode`, `selectedAddressId`, `selectedPaymentMethodId`, `requiresNewShipping` y `requiresNewPayment`

## 5. Implementación

Implementacion validada en `CheckoutPage.jsx`:

- carga direcciones guardadas y metodos guardados
- cachea esos recursos por usuario en `localStorage`
- detecta defaults con `getDefaultCheckoutItem`
- selecciona una sola fuente de verdad por tipo de dato:
  - `shippingMode` / `selectedAddressId`
  - `paymentMode` / `selectedPaymentMethodId`
- decide si crear o reutilizar con:
  - `requiresNewShipping`
  - `requiresNewPayment`
- en `handleSubmit`:
  - si hay seleccion existente, usa IDs ya seleccionados
  - si no, crea recurso nuevo y usa el ID retornado

Esta implementacion satisface exactamente la regla critica del prompt: no mezclar datos seleccionados con datos del formulario.

## 8. Riesgos

Riesgos residuales detectados:

- la implementacion depende de que `shippingApi` y `paymentApi` sigan devolviendo shapes compatibles con los normalizadores actuales
- el caching local de opciones de checkout puede quedar desactualizado si cambia el backend fuera de sesion, aunque el flujo ya intenta refrescar remotamente
- los E2E de reuse dependen de disponibilidad de backend/frontend de prueba y de que el dataset permita crear recursos sin rate limiting severo

Mitigacion actual:

- existe fallback manual
- existe cobertura unitaria y E2E real
- la carga remota esta envuelta con manejo de error visible
